Why XR Blocks?

Designed for rapid iteration, XR Blocks layers an approachable SDK on top of three.js so you can focus on behavior, not boilerplate. It targets Chrome with WebXR support on Android XR (e.g., Galaxy XR) and includes a full-featured desktop simulator to keep teams productive.

Hand Tracking & Gestures

Toggle options.enableGestures() to tap into prebuilt pinch, open-palm, fist, thumbs-up, point, and spread detections with gesture lifecycle events.

World Understanding

Blend depth sensing, geometry-aware physics, and object recognition. Pair it with Gemini for scene reasoning and live conversation loops.

Cross-Platform Delivery

Build once, deploy to XR devices or Chrome desktop simulator without rewriting interaction logic.

Community Driven

Maintained by XR Labs and collaborators; contributions welcome for demos, depth-based interactions, visual programming hooks, and more.

Usage Example

Import XR Blocks with an import map and build an interactive scene with just a few lines of code. Pinch in XR or click in the desktop emulator to change the object's color: 

<script type="module">
import * as THREE from 'three';
import * as xb from 'xrblocks';

class MainScript extends xb.Script {
  init() {
    this.add(new THREE.HemisphereLight(0xffffff, 0x666666, 3));
    const geometry = new THREE.CylinderGeometry(0.2, 0.2, 0.4, 32);
    const material = new THREE.MeshPhongMaterial({
      color: 0xffffff,
      transparent: true,
      opacity: 0.8,
    });
    this.player = new THREE.Mesh(geometry, material);
    this.player.position.set(0, xb.user.height - 0.5, -xb.user.objectDistance);
    this.add(this.player);
  }

  onSelectEnd() {
    this.player.material.color.set(Math.random() * 0xffffff);
  }
}

document.addEventListener('DOMContentLoaded', () => {
  xb.add(new MainScript());
  xb.init(new xb.Options());
});
</script>

Development Guide

Spin up XR Blocks locally in minutes:

  1. Clone: git clone --depth=1 git@github.com:google/xrblocks.git
  2. Install dependencies: npm ci
  3. Build distributable: npm run build
  4. Lint with ESLint: npm run lint
  5. Format with Prettier: npm run format

Recommended extensions: ESLint + Prettier in VS Code with Prettier set as the default formatter.

User Data & Permissions

WebXR, LiteRT hand tracking, depth sensing, and gesture recognition all process data locally on device. Denying permissions may limit feature behavior. Gemini-based AI features route data to Gemini servers— follow their Privacy & Terms when deploying.

Critical Security Rules
  • Never commit API keys or expose them in client bundles.
  • Keep Gemini API keys secret—treat them like passwords tied to quota and billing.
  • Use environment variables or secure vaults in production.
Uninstallation

Remove XR Blocks by deleting its importmap entries, JS imports, or running npm uninstall xrblocks inside your project.
